DEPENDENCY STRENGTH SCORING (1-5 Scale)
════════════════════════════════════════════════════════════════════

**5 - CRITICAL**: Appraisal completion → Loan approval (legal/policy requirement)
**4 - STRONG**: Risk rating assignment → Loan committee approval threshold (process blocker)
**3 - MODERATE**: DSCR calculation → Covenant compliance test (credit best practice)
**2 - WEAK**: Historical financial data → Current underwriting model (reference)
**1 - MINIMAL**: General credit policy rule → Specific loan type rule

**DEPENDENCY THRESHOLD: Only document dependencies rated ≥ 3 (MODERATE or higher)**

════════════════════════════════════════════════════════════════════
DEPENDENCY TYPES
════════════════════════════════════════════════════════════════════

1. **prerequisite** - Rule A MUST be satisfied before Rule B can execute
   Examples:
   - Appraisal completion → CRE loan approval (Strength: 5)
   - Beneficial ownership certification → New entity account opening (Strength: 5)
   - Environmental Phase I assessment → CRE collateral acceptance above $250K (Strength: 4)

2. **sequential** - Rules must execute in specific order for business process
   Examples:
   - Application → Credit analysis → Risk rating → Approval → Documentation → Closing (Strength: 4)
   - Origination → Annual review → Renewal → Modification (Strength: 3)

3. **conditional** - Rule B only applies when condition in Rule A is met
   Examples:
   - IF loan > $500,000 THEN global cash flow analysis required (Strength: 5)
   - IF CRE concentration > 300% risk-based capital THEN enhanced monitoring required (Strength: 5)
   - IF risk grade 6 or worse THEN special assets review within 30 days (Strength: 5)

4. **complementary** - Rules work together for comprehensive compliance
   Examples:
   - Multiple documentation rules for complete closing package (Strength: 4)
   - DSCR covenant + leverage covenant for full financial maintenance package (Strength: 4)

5. **contradictory** - Rules that may conflict
   Examples:
   - Bank credit policy DSCR floor vs OCC guidance minimum (Strength: 4)
   - State usury law vs federal rate floor for specific loan product (Strength: 3)

6. **override** - Rule B supersedes Rule A under specific conditions
   Examples:
   - SBA SOP requirements override bank standard policy for SBA loans (Strength: 5)
   - Regulatory minimum capital requirement overrides internal target when higher (Strength: 5)

7. **validation** - Rule B verifies output of Rule A
   Examples:
   - Loan review validates credit officer's risk grade assignment (Strength: 3)
   - Compliance officer review validates Reg O pre-approval determination (Strength: 4)
