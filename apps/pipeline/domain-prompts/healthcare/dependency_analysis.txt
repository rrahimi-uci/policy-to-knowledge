DEPENDENCY STRENGTH SCORING (1-5 Scale)
════════════════════════════════════════════════════════════════════

**5 - CRITICAL**: Patient authorization → PHI access (legal requirement)
**4 - STRONG**: Credentialing completion → Clinical privilege grant (process blocker)
**3 - MODERATE**: Admission H&P → Surgical procedure order (clinical best practice)
**2 - WEAK**: Prior hospitalization data → Current risk assessment (reference)
**1 - MINIMAL**: General policy rule → Specific clinical procedure rule

**DEPENDENCY THRESHOLD: Only document dependencies rated ≥ 3 (MODERATE or higher)**

════════════════════════════════════════════════════════════════════
DEPENDENCY TYPES
════════════════════════════════════════════════════════════════════

1. **prerequisite** - Rule A MUST be satisfied before Rule B can execute
   Examples:
   - Patient consent → Elective procedure (Strength: 5)
   - NPDB query → Credentialing approval (Strength: 5)
   - Prior authorization → Elective admission (Strength: 4)

2. **sequential** - Rules must execute in specific order for business process
   Examples:
   - Admission → Assessment → Care plan → Treatment → Discharge (Strength: 4)
   - Application → Credentialing review → Privilege grant (Strength: 3)

3. **conditional** - Rule B only applies when condition in Rule A is met
   Examples:
   - IF readmission within 30 days THEN enhanced review required (Strength: 5)
   - IF PHI breach ≥500 THEN media notification required (Strength: 5)
   - IF surgical procedure THEN universal protocol required (Strength: 5)

4. **complementary** - Rules work together for comprehensive compliance
   Examples:
   - Multiple documentation rules for complete medical record (Strength: 4)
   - HIPAA privacy + security rules for full PHI protection (Strength: 4)

5. **contradictory** - Rules that may conflict
   Examples:
   - Federal HIPAA vs more restrictive state privacy law (Strength: 4)
   - CMS CoP vs TJC standard with different timeframes (Strength: 3)

6. **override** - Rule B supersedes Rule A under specific conditions
   Examples:
   - State law overrides federal minimum when more protective (Strength: 5)
   - Emergency exception overrides standard consent requirement (Strength: 5)

7. **validation** - Rule B verifies output of Rule A
   Examples:
   - QA review validates initial coding (Strength: 3)
   - Peer review validates clinical documentation (Strength: 4)
