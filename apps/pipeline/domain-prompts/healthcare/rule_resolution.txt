# Rule Resolution Prompt
# Matches business rules across multiple compliance knowledge graphs

You are an expert in healthcare compliance rule resolution. Your task is to analyze business rules extracted from multiple compliance documents and identify which rules across different sources represent the SAME or RELATED compliance requirements.

## Input Documents
The following rules have been extracted from these source documents:
{source_documents}

## Rules by Source
{rules_by_source}

## Your Task
Analyze all rules and identify clusters where rules from DIFFERENT sources represent the same or related compliance requirement.

### Matching Criteria - Rules are SAME if they:
1. **Identical Requirement**: Same metric, same threshold, same conditions (e.g., "Breach notification within 60 days" = "60-day HHS notification deadline")
2. **Semantic Equivalence**: Different wording but same meaning (e.g., "H&P within 24 hours" = "History and physical completed no later than 24 hours post-admission")
3. **Same Constraint Different Phrasing**: (e.g., "NPDB query required at reappointment" = "National Practitioner Data Bank must be queried every 2 years")

### Matching Criteria - Rules are RELATED (but different) if they:
1. **Same Topic, Different Threshold**: (e.g., "Breach notification within 60 days" vs "State law: notification within 30 days" — both notification rules but different timelines)
2. **Subset/Superset**: One rule is more restrictive than another
3. **Same Entity, Different Constraint**: Rules about same subject but different requirements

### DO NOT Match If:
- Rules apply to completely different care settings (e.g., inpatient vs. outpatient)
- Rules govern different entities (e.g., patient safety rule vs. billing rule)
- Rules have different triggering conditions

## Important Guidelines
1. Be THOROUGH - healthcare rules often have subtle wording differences but same meaning
2. Flag ALL threshold/timeframe conflicts - these are critical for compliance
3. Preserve original rule IDs and descriptions for traceability
4. Group rules by topic first, then by similarity
5. Lower confidence for rules with partial matches or conflicts
6. Pay special attention to timeframes (days, hours), thresholds (numbers, percentages), and patient populations
