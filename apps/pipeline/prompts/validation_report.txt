KNOWLEDGE GRAPH QUALITY SCORING FRAMEWORK (0-100 Scale)
════════════════════════════════════════════════════════════════════

Assess knowledge graph across 6 dimensions:

**1. COMPLETENESS (0-25 points)**
- Source document coverage: 10 pts (% of documents processed)
- Entity extraction coverage: 8 pts (expected entities found vs missed)
- Relationship capture: 7 pts (critical relationships documented)

Scoring Guide:
- 23-25 pts: ≥95% coverage, minimal gaps
- 18-22 pts: 85-94% coverage, minor gaps
- 13-17 pts: 70-84% coverage, noticeable gaps
- <13 pts: <70% coverage, significant gaps

**2. CORRECTNESS (0-25 points)**
- Entity definition accuracy: 10 pts (entities match source material)
- Relationship accuracy: 8 pts (dependencies correctly identified)
- Business rule accuracy: 7 pts (rules accurately reflect requirements)

Scoring Guide:
- 23-25 pts: ≥98% accuracy, negligible errors
- 18-22 pts: 95-97% accuracy, minor errors
- 13-17 pts: 90-94% accuracy, some errors
- <13 pts: <90% accuracy, significant errors

**3. CONSISTENCY (0-20 points)**
- Naming conventions adherence: 8 pts (UPPER_CASE_WITH_UNDERSCORES, snake_case)
- Structural consistency: 7 pts (similar entities have similar attributes)
- Terminology consistency: 5 pts (same concept = same name throughout)

Scoring Guide:
- 18-20 pts: ≥95% consistency
- 14-17 pts: 85-94% consistency
- 10-13 pts: 70-84% consistency
- <10 pts: <70% consistency

**4. DEPENDENCY QUALITY (0-15 points)**
- Dependency coverage: 6 pts (critical dependencies documented)
- Strength scoring accuracy: 5 pts (strength scores justified)
- Circular dependency resolution: 4 pts (circular dependencies detected/resolved)

Scoring Guide:
- 14-15 pts: Comprehensive, accurate dependency mapping
- 11-13 pts: Good coverage, minor gaps
- 8-10 pts: Moderate coverage, some gaps
- <8 pts: Poor coverage, significant gaps

**5. USABILITY (0-10 points)**
- Attribute richness: 4 pts (entities have 5-10 useful attributes)
- Relationship clarity: 3 pts (relationship types are meaningful)
- Documentation quality: 3 pts (descriptions, examples, rationales provided)

Scoring Guide:
- 9-10 pts: Highly usable, rich documentation
- 7-8 pts: Good usability, adequate documentation
- 5-6 pts: Moderate usability, sparse documentation
- <5 pts: Poor usability, minimal documentation

**6. OPTIMIZATION (0-5 points)**
- Deduplication effectiveness: 3 pts (duplicates identified and merged)
- Data efficiency: 2 pts (no redundant entities/relationships)

Scoring Guide:
- 5 pts: Excellent optimization, minimal redundancy
- 3-4 pts: Good optimization, some redundancy
- 1-2 pts: Poor optimization, significant redundancy
- 0 pts: No optimization performed

**OVERALL QUALITY RATING:**
- 90-100: **Excellent** - Production-ready, minimal improvements needed
- 75-89: **Good** - Suitable for use, targeted improvements recommended
- 60-74: **Acceptable** - Usable with caution, notable improvements needed
- Below 60: **Poor** - Significant rework required before deployment

VALIDATION ANALYSIS FRAMEWORK
════════════════════════════════════════════════════════════════════

**COMPLETENESS ANALYSIS**
1. Calculate document coverage: (documents_processed / total_documents) × 100
2. Identify missed entities: Compare extracted entities to expected domain entities
3. Identify missing relationships: Check for orphaned entities (no relationships)
4. Assess rule coverage: Ensure all regulatory areas covered

**CORRECTNESS ANALYSIS**
1. Sample validation: Spot-check 10-20 random entities against source documents
2. Dependency validation: Verify dependency logic makes sense
3. Business rule validation: Check rules match regulatory requirements
4. Identify factual errors: Flag incorrect data, formulas, or references

**CONSISTENCY ANALYSIS**
1. Naming convention audit: Check all entities/attributes follow standards
2. Terminology audit: Flag instances where same concept uses different names
3. Structure audit: Identify entities with too few attributes or missing critical fields
4. Relationship type audit: Ensure relationship types used consistently

**DEPENDENCY QUALITY ANALYSIS**
1. Dependency coverage: Calculate (dependencies_found / expected_dependencies) × 100
2. Strength scoring audit: Verify strength scores align with scoring criteria
3. Circular dependency check: Confirm all circular dependencies detected and flagged
4. Conflict resolution: Verify contradictory dependencies have resolution strategies

**USABILITY ANALYSIS**
1. Attribute richness: Calculate average attributes per entity
2. Documentation completeness: Check for descriptions, examples, rationales
3. Relationship clarity: Assess if relationship types are specific and actionable
4. Navigation feasibility: Test if knowledge graph supports typical use cases

**OPTIMIZATION ANALYSIS**
1. Deduplication effectiveness: Calculate (duplicates_found / duplicates_merged)
2. Redundancy detection: Identify semantically identical entities with different names
3. Relationship efficiency: Check for redundant or overly generic relationships

REPORT QUALITY STANDARDS

✅ **Executive Summary Must Be Concise**
- 3-5 key strengths (specific, evidence-based)
- 3-5 key weaknesses (specific, actionable)
- Single primary recommendation (clear next step)
- Deployment readiness assessment (READY_WITH_IMPROVEMENTS | CONDITIONAL | NOT_READY)

✅ **Findings Must Be Evidence-Based**
❌ BAD: "Some entities are incomplete"
✅ GOOD: "29 entities (12%) have fewer than 5 attributes, average 3.2 attributes. Target: 5-10 attributes per entity."

✅ **Recommendations Must Be Actionable**
❌ BAD: "Improve data quality"
✅ GOOD: "Add 3 missing attributes to LOAN_APPROVAL_CRITERIA: approval_threshold, validation_method, regulatory_source. Reference source document 'Fannie Mae Selling Guide Section B3-3'."

✅ **Gap Analysis Must Identify Root Causes**
- Why was entity missed? (document not processed, extraction error, ambiguous text)
- Why is entity orphaned? (too generic, missing relationships, extraction error)
- What specific action resolves gap? (re-process document, add relationships, remove entity)

✅ **Improvement Roadmap Must Be Prioritized**
- HIGH: Blocks deployment or causes compliance violations (fix immediately)
- MEDIUM: Reduces usability or completeness (fix before full rollout)
- LOW: Nice-to-have improvements (fix iteratively)

✅ **Circular Dependencies Must Have Resolution Strategies**
- Describe the conflict clearly
- Explain why it's problematic (workflow deadlock, ambiguity)
- Suggest concrete resolution (refactor rules, add entry point, manual review)
- Flag if requires manual policy decision
