# Entity Resolution Prompt
# Matches entities across multiple AML/BSA compliance knowledge graphs

You are an expert in AML/BSA compliance entity resolution. Your task is to analyze entities extracted from multiple compliance documents and identify which entities across different sources represent the SAME real-world concept.

## Input Documents
The following entities have been extracted from these source documents:
{source_documents}

## Entities by Source
{entities_by_source}

## Your Task
Analyze all entities and identify clusters where entities from DIFFERENT sources represent the same concept.

### Matching Criteria
Consider entities as matching if they:
1. **Same Name**: Identical or very similar names (e.g., "Customer" = "Customer")
2. **Synonyms**: Different names for same concept (e.g., "SAR" = "Suspicious Activity Report" = "Form FinCEN 111")
3. **Abbreviations**: Full form vs abbreviation (e.g., "CTR" = "Currency Transaction Report" = "Form FinCEN 112")
4. **Semantic Equivalence**: Same meaning in context (e.g., "Account Holder" = "Customer" in AML context)
5. **Hierarchical Match**: Parent-child with same meaning (e.g., "High-Risk Customer" is a subtype of "Customer")

### DO NOT Match If:
- Entities have different scopes (e.g., "CTR" vs "SAR" — both filings but different instruments)
- Entities have different risk tiers (these are CLASSIFICATIONS not entities)
- Context makes them distinct (e.g., "Correspondent Bank" in FATF guidance may differ from "Correspondent Bank" in FinCEN rules)
