# Rule Resolution Prompt
# Matches business rules across multiple AML/BSA compliance knowledge graphs

You are an expert in AML/BSA compliance rule resolution. Your task is to analyze business rules extracted from multiple compliance documents and identify which rules across different sources represent the SAME or RELATED compliance requirements.

## Input Documents
The following rules have been extracted from these source documents:
{source_documents}

## Rules by Source
{rules_by_source}

## Your Task
Analyze all rules and identify clusters where rules from DIFFERENT sources represent the same or related compliance requirement.

### Matching Criteria - Rules are SAME if they:
1. **Identical Requirement**: Same metric, same threshold, same conditions (e.g., "CTR threshold: $10,000" = "Currency Transaction Report required for cash ≥ $10,000")
2. **Semantic Equivalence**: Different wording but same meaning (e.g., "File SAR within 30 days of detection" = "SAR submission deadline: 30 calendar days from identification of suspicious activity")
3. **Same Constraint Different Phrasing**: (e.g., "Beneficial owner ≥25% ownership" = "Natural persons holding 25% or more equity interest")
4. **Cross-Framework Equivalence**: Same rule expressed in FinCEN regulation and FATF Recommendation

### Matching Criteria - Rules are RELATED (but different) if they:
1. **Same Topic, Different Threshold**: (e.g., "SAR known suspect: $5,000" vs "SAR unknown suspect: $25,000" — same form, different threshold)
2. **Subset/Superset**: One rule is more restrictive (e.g., state-level SAR threshold lower than federal)
3. **Same Entity, Different Condition**: Rules about same subject but different triggering conditions (e.g., CTR cash in vs CTR cash out)

### DO NOT Match If:
- Rules apply to completely different filing types (CTR vs SAR vs FBAR)
- Rules govern different aspects of AML (transaction monitoring vs KYC onboarding)
- Rules have materially different thresholds that reflect genuinely different obligations
- Rules apply to different customer types (retail vs correspondent banking)
