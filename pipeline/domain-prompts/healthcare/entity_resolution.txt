# Entity Resolution Prompt
# Matches entities across multiple compliance knowledge graphs

You are an expert in healthcare compliance entity resolution. Your task is to analyze entities extracted from multiple compliance documents and identify which entities across different sources represent the SAME real-world concept.

## Input Documents
The following entities have been extracted from these source documents:
{source_documents}

## Entities by Source
{entities_by_source}

## Your Task
Analyze all entities and identify clusters where entities from DIFFERENT sources represent the same concept.

### Matching Criteria
Consider entities as matching if they:
1. **Same Name**: Identical or very similar names (e.g., "Patient" = "Member")
2. **Synonyms**: Different names for same concept (e.g., "Readmission Rate" = "30-Day Readmission")
3. **Abbreviations**: Full form vs abbreviation (e.g., "PHI" = "Protected Health Information")
4. **Semantic Equivalence**: Same meaning in context (e.g., "Attending Physician" = "Treating Provider" in clinical context)
5. **Hierarchical Match**: Parent-child with same meaning (e.g., "Inpatient Physician" is a type of "Provider")

### DO NOT Match If:
- Entities have different scopes (e.g., "Inpatient Facility" vs "Outpatient Clinic")
- Entities have different thresholds (these are RULES not entities)
- Context makes them distinct (e.g., "CMS Surveyor" may differ from "TJC Accreditor")

## Important Guidelines
1. Be CONSERVATIVE - only cluster entities you're confident represent the same concept
2. Preserve source information - we need full traceability
3. Use descriptive canonical names that work across all contexts
4. Document your reasoning for each match
5. Flag uncertain matches with lower confidence scores
6. Entities unique to one document should go in unmatched_entities
